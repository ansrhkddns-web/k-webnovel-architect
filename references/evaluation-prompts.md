# 평가 프롬프트 세트

이 문서는 스킬이 실제 요청에서 좋은 결과를 내는지 점검하기 위한 테스트 프롬프트 모음이다. 평가용 프롬프트는 정답을 알려주지 않고, 스킬이 스스로 장르 문법, 상업성, 회차 기능, 이탈 위험을 판단하게 만든다.

## 사용 원칙

- 프롬프트에는 기대 답안을 넣지 않는다.
- 장르, 소재, 목표만 제공한다.
- 결과물은 `output-quality-rubric.md`로 채점한다.
- 특정 실제 작품의 고유 설정이나 문장을 넣지 않는다.
- 테스트 후 부족한 항목은 스킬 문서나 참조 문서로 보강한다.

## 테스트 1. 초기 헌터물 기획

```text
$k-webnovel-architect 로 헌터물 신작을 상업 연재형으로 기획해줘.
주인공은 낮은 등급으로 무시받다가 남들이 못 보는 보상 조건을 발견한다.
제목 후보, 한 줄 콘셉트, 독자 페르소나, 성장 루프, 1~25화 로드맵, 유료화 후보까지 잡아줘.
```

필수 확인:
- 장르 명료도.
- 반복 보상 루프.
- 첫 5화 보상.
- 사회적 반응.
- 유료화 후보.

## 테스트 2. 로맨스판타지 진단

```text
$k-webnovel-architect 로 로맨스판타지 기획을 진단해줘.
여주인공은 악녀로 몰려 파혼당한 뒤, 가문 사업을 되살리며 전 약혼자와 궁정 세력에게 복수하려고 한다.
장르 문법, 관계 보상, 지위 상승, 1화 훅, 무료분 유지력, 유료화 지점을 봐줘.
```

필수 확인:
- 관계와 권력 사건의 결합.
- 악녀/파혼/사업 트로프 조합.
- 사교전 또는 평판 반전.
- 후회/복수 보상.

## 테스트 3. 1화 훅 개선

```text
$k-webnovel-architect 로 1화 훅을 개선해줘.
현재 1화는 세계관 설명으로 시작하고, 주인공의 특수 능력은 2화 후반에 나온다.
장르가 바로 보이도록 첫 화면, 첫 사건, 엔딩 훅을 다시 설계해줘.
```

필수 확인:
- 설명 삭제/후퇴 지시.
- 첫 화면 압박.
- 장르 신호 앞당김.
- 주인공 선택.
- 구체적 엔딩 훅.

## 테스트 4. 유료화 지점 설계

```text
$k-webnovel-architect 로 유료화 후보를 잡아줘.
무료분은 25화 안팎이고, 1부 초반 적대자를 무너뜨린 뒤 더 큰 조직이 드러나는 구조야.
어디서 유료화하면 좋은지 후보 3개와 유료분 초반 3화 목표를 설계해줘.
```

필수 확인:
- 직전 보상.
- 다음 약속.
- 나쁜 절단 회피.
- 유료분 초반 회수.

## 테스트 5. 기존 설정집 분석

```text
$k-webnovel-architect 로 기존 작품 바이블을 분석해줘.
개요, 세계관, 인물, 성장 시스템, 아크, 회차표, 원고 일부가 있어.
파일 구조, 회차별 기능, 반복 보상, 설정 충돌 위험, 더 고도화할 문서를 정리해줘.
```

필수 확인:
- 파일 구조 분석.
- 엔진 분류.
- 회차 기능 추출.
- 일반화 가능한 작업 방식과 고유 데이터 분리.

## 테스트 6. 제목/소개글/태그 생성

```text
$k-webnovel-architect 로 제목과 소개글을 만들어줘.
장르는 아카데미+빙의물이고, 주인공은 원작에서 죽는 엑스트라 교관에게 빙의한다.
제목 후보 15개, 20자 판매 문장, 150자 소개글, 태그 묶음을 만들어줘.
```

필수 확인:
- 제목 유형 분리.
- 장르/처지/우위/보상 신호.
- 소개글 과장 금지.
- 태그 충돌 점검.

## 테스트 7. 장기 시리즈 설계

```text
$k-webnovel-architect 로 200화 이상 갈 수 있는 시리즈 구조를 설계해줘.
초반은 개인 생존과 성장, 중반은 조직/거점 운영, 후반은 세계 질서 재편으로 확장하고 싶어.
시즌별 질문, 보상 사다리, 적대자 단계, 인플레이션 방지책을 정리해줘.
```

필수 확인:
- 개인->조직->사회->세계 확장.
- 보상 사다리 복수화.
- 적대자 압박 방식 변화.
- 초반 약속 유지.

## 테스트 8. 원고 리뷰와 수정 처방

```text
$k-webnovel-architect 로 1~10화 원고를 리뷰해줘.
문장 감상보다 상업 연재 기능을 봐줘.
장르 약속, 주인공 엔진, 보상 회전율, 1화 훅, 유료화 신뢰, 수정 우선순위를 표로 정리해줘.
```

필수 확인:
- 치명 문제 우선.
- 점수표.
- 수정 처방.
- 회차표 수정안.

## 테스트 9. 플랫폼 리서치 계획

```text
$k-webnovel-architect 로 플랫폼별 리서치 계획을 세워줘.
현판 신작을 준비 중이고, 문피아형/카카오페이지형/네이버시리즈형/리디형 패키징 차이를 확인하고 싶어.
최신 데이터는 직접 확인한다는 전제로 무엇을 봐야 하는지 체크리스트를 만들어줘.
```

필수 확인:
- 최신 데이터 고정 금지.
- 플랫폼별 확인 항목.
- 제목/태그/무료분/유료화 비교.

## 테스트 10. 회차 집필 지시서

```text
$k-webnovel-architect 로 12화 집필 지시서를 만들어줘.
11화에서 주인공의 작은 성장이 주변 인물에게 처음 의심받았고, 12화는 그 의심이 사회적 압박으로 바뀌는 화야.
필수 장면, 금지할 설명, 독자 보상, 엔딩 훅을 정리해줘.
```

필수 확인:
- 회차 기능.
- 필수 장면.
- 독자 보상.
- 쓰면 안 되는 장면.
- 엔딩 훅.
