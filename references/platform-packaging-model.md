# 플랫폼 패키징 모델

플랫폼별 최신 랭킹, 이벤트, 노출 정책은 변할 수 있으므로 사용자가 현재 시장 확인을 요구하면 반드시 최신 자료를 따로 확인한다. 이 문서는 특정 플랫폼의 현재 순위를 저장하지 않고, 플랫폼 성격별로 어떤 패키징 판단을 해야 하는지 정리한다.

## 플랫폼 성격별 판단

| 플랫폼 성격 | 강한 작품 | 패키징 핵심 | 위험 |
|---|---|---|---|
| 남성향 오픈 연재형 | 판타지, 현판, 무협, 스포츠, 전문직, 대체역사 | 제목에서 장르/우위/보상 즉시 노출 | 초반 보상 지연 시 이탈 빠름 |
| 대형 큐레이션/기다림형 | 판타지, 로판, 액션판타지, 회귀, 장기 성장물 | 소개글과 무료분의 큰 약속 | 무료분 끝 보상 부족 |
| 로맨스/여성향 강세형 | 로맨스, 로판, 비엘, 지엘, 관계 중심물 | 관계 약속, 지위 변화, 감정 훅 | 관계가 사건에 영향 없으면 약함 |
| 완결 몰아보기형 | 로판, 비엘, 미스터리, 전문직, 장편 판타지 | 장기 떡밥, 권별 완성감, 회수 신뢰 | 초반만 강하고 중후반 무너지면 불리 |
| 웹툰/IP 확장형 | 시각적 능력, 강한 관계, 명확한 무대, 반복 사건 | 장면성, 캐릭터 실루엣, 상징 공간 | 내면 독백 의존이 과함 |
| 공모전/신인 발굴형 | 장르 명료, 초반 훅 강함, 독자 반응 빠름 | 1화, 제목, 연참 지속성 | 설정만 크고 원고 생산성이 낮음 |

## 패키징 요소

| 요소 | 해야 할 일 |
|---|---|
| 제목 | 장르, 주인공 처지, 우위, 보상 중 2개 이상 노출 |
| 태그 | 주 장르, 보상, 장치, 관계, 톤을 구분 |
| 소개글 | 처지 -> 우위/사건 -> 반복 보상 -> 장기 위기 |
| 1화 첫 화면 | 압박 또는 이상성을 바로 보여줌 |
| 1화 엔딩 | 다음 화에서 확인할 구체적 질문 |
| 5화까지 | 반복 보상 구조 증명 |
| 무료분 끝 | 첫 큰 보상과 더 큰 위기 동시 제시 |
| 유료분 초반 | 결제 직후 보상 회수로 신뢰 유지 |

## 태그 조합

태그는 아래 묶음에서 고른다.

| 묶음 | 예시 |
|---|---|
| 장르 | 판타지, 현판, 무협, 로판, 로맨스, 비엘, 미스터리, 스포츠 |
| 장치 | 회귀, 빙의, 환생, 시스템, 상태창, 게이트, 탑, 아카데미, 계약 |
| 보상 | 성장, 먼치킨, 사이다, 복수, 생존, 힐링, 경영, 두뇌전 |
| 관계 | 혐관, 후회, 계약관계, 가족, 동료, 스승제자, 집착 |
| 무대 | 학교, 궁정, 회사, 병원, 법정, 연예계, 전장, 영지 |
| 톤 | 통쾌함, 어두움, 코미디, 피폐, 잔잔함, 하드보일드 |

태그가 서로 다른 독자 기대를 만들면 우선순위를 정한다. 예를 들어 `힐링`과 `피폐`가 함께 있으면 어느 쪽이 주 보상인지 소개글에서 명확히 해야 한다.

## 소개글 점검

좋은 소개글은 아래 질문에 답한다.

- 주인공은 어떤 처지인가?
- 어떤 비정상 사건 또는 우위가 생기는가?
- 독자는 무엇을 반복해서 보게 되는가?
- 장기적으로 어떤 더 큰 문제가 열리는가?
- 1화와 소개글의 약속이 같은가?

## 무료분/유료분 설계

```markdown
| 구간 | 목표 | 필수 보상 | 위험 |
|---|---|---|---|
| 1화 | 클릭 유지 | 압박, 이상성, 다음 질문 | 설명 과밀 |
| 2-5화 | 반복 재미 증명 | 첫 사용, 첫 보상 | 소재 미사용 |
| 6-10화 | 비용과 적대 강화 | 후폭풍, 첫 대가 | 같은 장면 반복 |
| 11-25화 | 결제 전 신뢰 | 중간 사이다, 관계/지위 변화 | 고구마 누적 |
| 유료분 초반 | 결제 보상 | 바로 이어지는 보상 회수 | 결제 후 늘어짐 |
```
