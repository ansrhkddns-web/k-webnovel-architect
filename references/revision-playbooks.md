# 수정 처방 플레이북

진단은 수정으로 이어져야 한다. 이 문서는 상업 웹소설 기획에서 자주 나오는 문제를 어떤 방식으로 고칠지 정리한다.

## 공통 처방 순서

1. 삭제보다 기능을 먼저 확인한다.
2. 장르 약속을 깨는 문제부터 고친다.
3. 초반 1-5화의 보상 지연을 우선 처리한다.
4. 주인공 선택이 없는 장면은 선택/대가를 추가한다.
5. 설정 설명은 장면 속 문제 해결로 바꾼다.
6. 고구마가 길면 작은 보상이나 중간 사이다를 삽입한다.
7. 인물 기능이 겹치면 통합하거나 역할을 바꾼다.

## 문제별 처방

| 문제 | 증상 | 처방 |
|---|---|---|
| 장르가 흐림 | 제목/1화에서 장르가 안 보임 | 장르 키워드, 핵심 소재, 보상 장면을 앞당긴다 |
| 1화가 늦음 | 세계 설명이 먼저 나옴 | 첫 장면을 압박/이상성으로 바꾼다 |
| 주인공이 수동적 | 사건이 남에 의해만 진행 | 거절, 도망, 실험, 거래, 기록 중 하나를 선택하게 한다 |
| 보상이 늦음 | 5화까지 변화가 없음 | 작은 성장, 정보, 관계 변화, 자원 획득을 넣는다 |
| 설정 과밀 | 이름과 규칙이 쏟아짐 | 현재 장면에 필요한 규칙만 남긴다 |
| 빌런이 평면적 | 그냥 나쁜 사람 | 겉명분, 실제 욕망, 주인공 약점 공격 방식을 준다 |
| 사이다가 약함 | 응징이 감정적으로 안 터짐 | 압박, 명분, 증거, 목격자, 결과를 쌓는다 |
| 로맨스가 장식 | 관계가 사건에 영향 없음 | 관계 선택이 목표/위험을 바꾸게 한다 |
| 성장 무체감 | 숫자만 오름 | 몸, 물건, 관계, 제도, 선택지로 체감시킨다 |
| 반복 피로 | 같은 장면이 계속됨 | 장소, 상대, 보상, 실패 이유, 주변 반응을 바꾼다 |
| 유료화 불신 | 보상 없이 끊음 | 직전 보상과 다음 약속을 겹친다 |
| 장기 떡밥 과잉 | 질문만 쌓임 | 작은 답과 실용적 단서를 먼저 준다 |

## 1화 수정 플레이북

```markdown
## 1화 수정
- 삭제할 설명:
- 앞당길 장르 신호:
- 강화할 주인공 압박:
- 추가할 선택/대가:
- 엔딩에 남길 구체물/위험:
```
