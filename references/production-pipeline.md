# 제작 파이프라인

상업 연재형 웹소설은 기획서 작성에서 끝나지 않는다. 콘셉트, 바이블, 회차표, 원고, 검수, 수정, 연재 전략이 이어지는 파이프라인이 필요하다.

## 전체 파이프라인

```text
아이디어
-> 콘셉트 검증
-> 장르/독자 계약
-> 작품 바이블
-> 초반 회차표
-> 1화 훅 설계
-> 원고 작성
-> 회차 기능 검수
-> 이탈 위험 수정
-> 유료화/연재 전략
-> 장기 연속성 관리
```

## 단계별 산출물

| 단계 | 산출물 | 통과 기준 |
|---|---|---|
| 콘셉트 | 한 줄 콘셉트, 제목 후보 | 10초 안에 장르와 보상이 보임 |
| 독자 계약 | 주 독자, 보상, 이탈 위험 | 5화 안에 줄 보상이 명확 |
| 바이블 | 세계관, 인물, 성장, 떡밥 | 50화 이상 반복 엔진 존재 |
| 회차표 | 1-25화 카드 | 각 화에 보상/변화/훅 존재 |
| 원고 | 실제 회차 | 장르 약속이 장면으로 구현 |
| 검수 | 진단표 | 치명 이탈 위험 없음 |
| 수정 | 리라이트 지시 | 문제 위치와 처방이 명확 |
| 연재 | 무료분/유료분 전략 | 결제 전 신뢰와 유료분 회수 |

## 회차 작성 전 체크

- 이번 화의 기능은 무엇인가?
- 시작 압박은 무엇인가?
- 주인공이 어떤 선택을 하는가?
- 독자가 받을 보상은 무엇인가?
- 이전 회차 떡밥을 상기하거나 회수하는가?
- 엔딩 훅은 구체적인가?

## 회차 작성 후 체크

- 보상/변화가 실제 장면으로 보이는가?
- 설명이 사건보다 앞서지 않았는가?
- 인물 행동이 기존 욕망과 맞는가?
- 새 설정이 기존 설정과 충돌하지 않는가?
- 다음 화를 누를 이유가 있는가?

## 연재 운영 체크

- 2-4화마다 작은 보상이 있는가?
- 8-12화마다 중간 사이다나 관계 변화가 있는가?
- 무료분 끝 전에 결제 신뢰를 만들었는가?
- 유료화 직후 1-3화 안에 보상을 회수하는가?
- 독자 반응에 맞춰 바꿀 수 있는 요소와 바꾸면 안 되는 장르 약속을 구분했는가?

## 작업 지시 출력

```markdown
## 다음 작업 지시

### 지금 할 일
-

### 작성할 문서
-

### 수정할 회차
-

### 검수 기준
-

### 다음 결정
-
```
