# 레퍼런스 리서치 프로토콜

상업 웹소설 기획에서 레퍼런스는 작품을 베끼기 위한 자료가 아니라, `시장에서 반복 검증된 보상 구조`를 추출하기 위한 자료다. 특정 작품의 고유명사, 사건, 문장, 설정값을 스킬이나 새 기획에 복사하지 않는다.

## 리서치 원칙

- 최신 랭킹, 플랫폼 정책, 공모전 조건, 장르 유행은 변하므로 필요할 때 새로 확인한다.
- 레퍼런스 작품명은 사용자 프로젝트 분석 결과에만 쓰고, 스킬 본문에는 저장하지 않는다.
- 추출 대상은 `장르 슬롯`, `제목 신호`, `초반 보상`, `유료화 훅`, `독자 반응 패턴`, `IP 확장 신호`다.
- 문장, 장면, 설정, 고유 인물 구도는 복사하지 않는다.
- 3개 이하 작품으로 일반화하지 않는다. 최소 10개, 가능하면 30-100개 단위로 패턴을 본다.
- 상위권 500개를 참고할 때도 개별 데이터가 아니라 빈도와 구조만 추출한다.

## 리서치 소스 유형

| 소스 | 확인할 것 | 주의 |
|---|---|---|
| 플랫폼 장르 페이지 | 장르 분류, 태그, 제목 신호 | 현재 노출 순위는 변한다 |
| 랭킹/베스트 페이지 | 상위권 제목 패턴, 장르 집중도 | 이벤트/프로모션 영향을 구분한다 |
| 공모전 수상작 목록 | 신인 시장에서 먹히는 콘셉트 | 공모전 주제와 심사 기준을 확인한다 |
| 출판사/플랫폼 보도자료 | IP 확장, 웹툰화/영상화 경향 | 홍보 문구를 그대로 믿지 않는다 |
| 독자 리뷰/별점 | 이탈 원인, 기대 보상 | 극단 리뷰만 보지 않는다 |
| 앱스토어 설명 | 플랫폼이 스스로 강조하는 장르/태그 | 마케팅 문구와 실제 랭킹은 다를 수 있다 |
| 산업 보고서 | 시장 크기, 수익모델, 장르 범위 | 발행 연도와 조사 범위를 확인한다 |

## 상위권 다수 분석 절차

1. 같은 플랫폼과 같은 장르 슬롯에서 표본을 모은다.
2. 제목을 먼저 분류한다: 키워드형, 상황형, 능력형, 관계형, 모순형.
3. 소개글 첫 문장이 무엇을 약속하는지 분류한다.
4. 무료분 초반에서 어떤 보상을 가장 먼저 주는지 본다.
5. 반복 사건 엔진을 찾는다.
6. 보상 사다리가 무엇인지 분류한다.
7. 유료화 직전 또는 큰 국면 전환 직전의 훅 유형을 찾는다.
8. 독자 리뷰에서 반복되는 칭찬과 불만을 분류한다.
9. 개별 사례를 버리고 장르별 공통 패턴만 남긴다.

## 분석 금지

- 특정 작품의 설정 조합을 그대로 새 작품에 옮기지 않는다.
- 고유한 반전, 인물 관계, 문장 리듬, 회차 사건을 가져오지 않는다.
- 리뷰 문장을 장문 인용하지 않는다.
- 단일 플랫폼의 현재 랭킹을 전체 시장 진실로 취급하지 않는다.
- 해외 플랫폼 유행을 국내 장르 독자 기대와 동일시하지 않는다.
