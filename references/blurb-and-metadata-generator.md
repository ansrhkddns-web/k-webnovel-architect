# 소개글과 메타데이터 생성기

상업 웹소설의 제목, 소개글, 태그, 한 줄 판매 문장은 본문보다 먼저 독자를 만난다. 이 문서는 작품 패키징 산출물을 만들기 위한 기준이다.

## 제목 후보 세트

제목은 한 방향만 만들지 않는다.

| 유형 | 목표 |
|---|---|
| 안정형 | 장르와 보상을 명확히 노출 |
| 후킹형 | 모순이나 강한 상황으로 클릭 유도 |
| 차별화형 | 기억에 남는 twist 강조 |
| 플랫폼형 | 검색/태그 친화적 |
| IP형 | 웹툰/영상화 시에도 어색하지 않음 |

## 소개글 공식

```text
[현재 처지]였던 주인공은 [비정상 사건/특수 우위]를 얻는다.
[작은 목표]를 위해 움직이던 그는 [반복 사건]을 통해 [보상]을 쌓아간다.
하지만 [장기 위험/더 큰 적]이 드러나며, [더 큰 약속]이 시작된다.
```

## 소개글 길이별 산출물

```markdown
### 20자 판매 문장
-

### 60자 소개 문장
-

### 150자 소개글
-

### 400자 상세 소개글
-
```

## 태그 출력

```markdown
## 태그
- 장르:
- 장치:
- 보상:
- 관계:
- 무대:
- 톤:
```

## 금지

- 본문에서 5화 안에 증명하지 못할 보상을 소개글에 쓰지 않는다.
- 장르 독자를 속이는 태그를 넣지 않는다.
- 너무 많은 태그로 서로 다른 기대를 만들지 않는다.
- 결말급 반전을 소개글에 설명하지 않는다.

## 패키징 출력

```markdown
## 패키징

### 제목 후보
#### 안정형
-

#### 후킹형
-

#### 차별화형
-

### 추천 제목
-

### 소개글
-

### 태그
-

### 패키징 위험
-
```
